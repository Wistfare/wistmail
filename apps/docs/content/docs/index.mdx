Wistfare Mail provides a simple API to send transactional emails from your applications. Domains, templates, and analytics are managed through the Wistfare Mail dashboard — the API is focused on what your code needs.

## What the API Does

| Feature | Description |
|---------|-------------|
| **Send Emails** | Single and batch transactional email delivery with DKIM signing |
| **Webhooks** | Real-time notifications for delivery, bounce, open, and click events |
| **Audiences** | Manage contact lists and subscriptions for targeted sending |

## Quick Example

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript
import { WistMail } from 'wistmail'

const wm = new WistMail({ apiKey: 'wm_your_api_key' })

await wm.emails.send({
  from: 'hello@yourdomain.com',
  to: 'user@example.com',
  subject: 'Welcome!',
  html: '<h1>Welcome to our platform</h1>',
})
```
</Tab>
<Tab value="Python">
```python
from wistmail import WistMail

wm = WistMail(api_key="wm_your_api_key")

wm.emails.send(
    from_address="hello@yourdomain.com",
    to="user@example.com",
    subject="Welcome!",
    html="<h1>Welcome to our platform</h1>",
)
```
</Tab>
<Tab value="cURL">
```bash
curl -X POST https://your-wistfare.com/api/v1/emails \
  -H "X-API-Key: wm_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "hello@yourdomain.com",
    "to": "user@example.com",
    "subject": "Welcome!",
    "html": "<h1>Welcome to our platform</h1>"
  }'
```

## Authentication

All API requests require an API key sent via the `X-API-Key` header:

```
X-API-Key: wm_your_api_key
```

Create API keys from **Settings > API Keys** in the Wistfare Mail dashboard. Each key has scopes that control access:

| Scope | Access |
|-------|--------|
| `emails:send` | Send single and batch emails |
| `emails:read` | Read email delivery status |
| `contacts:manage` | Manage audiences and contacts |
| `webhooks:manage` | Configure webhook endpoints |

## Rate Limits

- **Default**: 10 requests/second per API key
- **Batch endpoints**: 5 requests/second

Rate limit headers are included in every response:
```
X-RateLimit-Limit: 10
X-RateLimit-Remaining: 7
X-RateLimit-Reset: 1711584000
```

## Error Format

All errors return a consistent JSON structure:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid email address"
  }
}
```

| Status | Code | Description |
|--------|------|-------------|
| 400 | `VALIDATION_ERROR` | Invalid request body |
| 401 | `UNAUTHORIZED` | Missing or invalid API key |
| 403 | `INSUFFICIENT_SCOPE` | API key lacks required scope |
| 404 | `NOT_FOUND` | Resource does not exist |
| 429 | `RATE_LIMITED` | Too many requests |
| 500 | `INTERNAL_ERROR` | Server error |
