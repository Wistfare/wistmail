All API requests require a valid API key sent as a Bearer token in the `Authorization` header.

## API Keys

```
Authorization: Bearer wm_your_api_key
```

API keys are prefixed with `wm_` and are 64+ characters long. The key is shown **once** at creation time — store it securely.

## Scopes

Each API key has scopes that control what it can access:

| Scope | Access |
|-------|--------|
| `emails:send` | Send single and batch emails |
| `emails:read` | Read email status and delivery info |
| `domains:manage` | Create, verify, and delete domains |
| `templates:manage` | CRUD email templates |
| `contacts:manage` | Manage audiences and contacts |
| `webhooks:manage` | Configure webhook endpoints |
| `analytics:read` | Read delivery and engagement analytics |

## Creating API Keys

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript

// The SDK uses an existing API key for all operations

```
</Tab>
<Tab value="Python">

<Tab value="cURL">
```bash
# Create a key with specific scopes (requires session auth)
curl -X POST https://your-wistmail.com/api/v1/api-keys \
  -H "Cookie: wm_session=your_session" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Sending Key",
    "scopes": ["emails:send", "emails:read"]
  }'
```

Response:
```json

  "name": "Production Sending Key",

  "keyPrefix": "wm_a1b2",
  "scopes": ["emails:send", "emails:read"],
  "createdAt": "2026-03-28T00:00:00Z"
}
```

## Rate Limiting

API requests are rate limited per API key:

- **Default**: 10 requests/second
- **Batch endpoints**: 5 requests/second

Rate limit headers are included in every response:

```
X-RateLimit-Limit: 10
X-RateLimit-Remaining: 7
X-RateLimit-Reset: 1711584000
```

When the limit is exceeded, the API returns `429 Too Many Requests` with a `Retry-After` header.

## Error Responses

All errors follow a consistent format:

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid API key"
  }
}
```

| Status | Code | Description |
|--------|------|-------------|
| 400 | `VALIDATION_ERROR` | Invalid request body or parameters |
| 401 | `UNAUTHORIZED` | Missing or invalid API key |
| 403 | `INSUFFICIENT_SCOPE` | API key lacks required scope |
| 404 | `NOT_FOUND` | Resource does not exist |
| 429 | `RATE_LIMITED` | Too many requests |
| 500 | `INTERNAL_ERROR` | Server error |
