---
title: cURL Reference
description: Complete HTTP API reference with cURL examples for all WistMail endpoints.
---

All API requests use the base URL of your WistMail instance and require a Bearer token for authentication.

```bash
curl https://your-wistmail.com/api/v1/{endpoint} \
  -H "Authorization: Bearer wm_your_api_key" \
  -H "Content-Type: application/json"
```

## Emails

### Send an Email

`POST /api/v1/emails` -- Scope: `emails:send`

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript
const { id } = await wm.emails.send({
  from: 'hello@yourdomain.com',
  to: ['user@example.com'],
  subject: 'Welcome!',
  html: '<h1>Hello!</h1>',
})
```
</Tab>
<Tab value="Python">
```python
result = wm.emails.send(
    from_address="hello@yourdomain.com",
    to=["user@example.com"],
    subject="Welcome!",
    html="<h1>Hello!</h1>",
)
```
</Tab>
<Tab value="cURL">
```bash
curl -X POST https://your-wistmail.com/api/v1/emails \
  -H "Authorization: Bearer wm_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "hello@yourdomain.com",
    "to": ["user@example.com"],
    "subject": "Welcome!",
    "html": "<h1>Hello!</h1>",
    "tags": { "campaign": "onboarding" }
  }'
```
</Tab>
</Tabs>

**Response** `200 OK`:
```json
{ "id": "eml_abc123" }
```

### Batch Send

`POST /api/v1/emails/batch` -- Scope: `emails:send`

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript
const { ids } = await wm.emails.batchSend([
  { from: 'hello@yourdomain.com', to: 'a@example.com', subject: 'Hi A', html: '<p>Hi</p>' },
  { from: 'hello@yourdomain.com', to: 'b@example.com', subject: 'Hi B', html: '<p>Hi</p>' },
])
```
</Tab>
<Tab value="Python">
```python
result = wm.emails.batch_send([
    {"from": "hello@yourdomain.com", "to": "a@example.com", "subject": "Hi A", "html": "<p>Hi</p>"},
    {"from": "hello@yourdomain.com", "to": "b@example.com", "subject": "Hi B", "html": "<p>Hi</p>"},
])
```
</Tab>
<Tab value="cURL">
```bash
curl -X POST https://your-wistmail.com/api/v1/emails/batch \
  -H "Authorization: Bearer wm_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "emails": [
      {
        "from": "hello@yourdomain.com",
        "to": "a@example.com",
        "subject": "Hi A",
        "html": "<p>Hi</p>"
      },
      {
        "from": "hello@yourdomain.com",
        "to": "b@example.com",
        "subject": "Hi B",
        "html": "<p>Hi</p>"
      }
    ]
  }'
```
</Tab>
</Tabs>

**Response** `200 OK`:
```json
{ "ids": ["eml_abc123", "eml_def456"] }
```

## Error Responses

All errors follow a consistent JSON format:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid email address format"
  }
}
```

| Status | Code | Description |
|--------|------|-------------|
| 400 | `VALIDATION_ERROR` | Invalid request body or parameters |
| 401 | `UNAUTHORIZED` | Missing or invalid API key |
| 403 | `INSUFFICIENT_SCOPE` | API key lacks required scope |
| 404 | `NOT_FOUND` | Resource does not exist |
| 429 | `RATE_LIMITED` | Too many requests (check `Retry-After` header) |
| 500 | `INTERNAL_ERROR` | Server error |
