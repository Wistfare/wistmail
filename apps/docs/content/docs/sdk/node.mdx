## Configuration

```typescript
import { WistMail } from 'wistmail'

const wm = new WistMail({
  apiKey: 'wm_your_api_key',       // Required
  baseUrl: 'https://your-wistmail.com', // Default: https://api.wistmail.com
  timeout: 30000,                   // Default: 30000ms
})
```

| Option | Type | Required | Default | Description |
|--------|------|----------|---------|-------------|
| `apiKey` | string | Yes | -- | Your WistMail API key |
| `baseUrl` | string | No | `https://api.wistmail.com` | WistMail server URL |
| `timeout` | number | No | `30000` | Request timeout in milliseconds |

## Emails

### `wm.emails.send(params)`

Send a single email.

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript
const { id } = await wm.emails.send({
  from: 'hello@yourdomain.com',
  to: ['user@example.com'],
  subject: 'Welcome!',
  html: '<h1>Hello!</h1>',
  tags: { campaign: 'onboarding' },
})
```
</Tab>
<Tab value="Python">
```python
result = wm.emails.send(
    from_address="hello@yourdomain.com",
    to=["user@example.com"],
    subject="Welcome!",
    html="<h1>Hello!</h1>",
    tags={"campaign": "onboarding"},
)
```
</Tab>
<Tab value="cURL">
```bash
curl -X POST https://your-wistmail.com/api/v1/emails \
  -H "Authorization: Bearer wm_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "hello@yourdomain.com",
    "to": ["user@example.com"],
    "subject": "Welcome!",
    "html": "<h1>Hello!</h1>",
    "tags": { "campaign": "onboarding" }
  }'
```
</Tab>
</Tabs>

**Returns**: `{ id: string }`

### `wm.emails.batchSend(emails)`

Send up to 100 emails in a single request.

<Tabs items={["Node.js", "Python", "cURL"]}>
<Tab value="Node.js">
```typescript
const { ids } = await wm.emails.batchSend([
  { from: 'hello@yourdomain.com', to: 'a@example.com', subject: 'Hi A', html: '<p>Hi</p>' },
  { from: 'hello@yourdomain.com', to: 'b@example.com', subject: 'Hi B', html: '<p>Hi</p>' },
])
```
</Tab>
<Tab value="Python">
```python
result = wm.emails.batch_send([
    {"from": "hello@yourdomain.com", "to": "a@example.com", "subject": "Hi A", "html": "<p>Hi</p>"},
    {"from": "hello@yourdomain.com", "to": "b@example.com", "subject": "Hi B", "html": "<p>Hi</p>"},
])
```
</Tab>
<Tab value="cURL">
```bash
curl -X POST https://your-wistmail.com/api/v1/emails/batch \
  -H "Authorization: Bearer wm_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "emails": [
      {"from": "hello@yourdomain.com", "to": "a@example.com", "subject": "Hi A", "html": "<p>Hi</p>"},
      {"from": "hello@yourdomain.com", "to": "b@example.com", "subject": "Hi B", "html": "<p>Hi</p>"}
    ]
  }'
```
</Tab>
</Tabs>

**Returns**: `{ ids: string[] }`

## Error Handling

The SDK throws typed errors that you can catch:

```typescript
import { WistMailError, ValidationError, RateLimitError, NotFoundError } from 'wistmail'

try {
  await wm.emails.send({ /* ... */ })
} catch (err) {
  if (err instanceof ValidationError) {
    console.error('Invalid request:', err.message, err.details)
  } else if (err instanceof RateLimitError) {
    console.error(`Rate limited. Retry after ${err.retryAfter}s`)
  } else if (err instanceof NotFoundError) {
    console.error('Resource not found:', err.message)
  } else if (err instanceof WistMailError) {
    console.error(`Error ${err.statusCode}: ${err.message}`)
  }
}
```

| Error Class | HTTP Status | Description |
|-------------|-------------|-------------|
| `AuthenticationError` | 401 | Invalid or missing API key |
| `ValidationError` | 400 | Invalid request parameters |
| `NotFoundError` | 404 | Resource does not exist |
| `RateLimitError` | 429 | Too many requests |
| `WistMailError` | Other | Base error class for all errors |
